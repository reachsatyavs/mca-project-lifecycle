Change Requests via CR Process

How to handle scope changes in the Deliver phase

By Semester 3, the requirements are well-established. Any new requirement or change must be submitted as a Change Request (CR) — evaluated for impact before being accepted or deferred.

CR Evaluation Criteria in Semester 3

QuestionExample Answer (Food Delivery)
Is this change truly necessary for the system to function?CR: "Add promo code support" → Not necessary; defer to Semester 4 enhancement log
Does it affect existing tested functionality?If yes, all affected tests must be rerun after the change
Can it be implemented in ≤ 3 days?If it takes longer, it risks destabilising the Semester 3 delivery timeline
Is it part of a P1 bug fix?Bug fixes are always approved; they are not CRs

Semester 3 Rule: Freeze Feature Scope

In Semester 3, the goal is to complete and test what was already planned — not to add new features. Any feature not in the approved backlog is deferred to Semester 4's Enhancement Log. This discipline is what makes projects actually get finished.

New Functional Requirements — FR-8xx (AI Agent)

FR-8xx — FoodBot Agentic AI
FR-801 · Customer Support Chat
The system shall provide a chat widget on the customer portal where users can type natural language queries about their orders, the restaurant, or delivery. The AI agent shall respond within 5 seconds on standard hardware.
FR-802 · Order Status via Agent
The agent shall be able to retrieve real-time order status, delivery partner location estimate, and expected delivery time by calling the get_order_status tool — without the customer navigating away from the chat.
FR-803 · Issue Resolution with Compensation
When a customer reports a delivery problem (cold food, missing item, wrong order), the agent shall investigate the order, assess the severity, and apply a proportional wallet credit (₹50–₹200) using the resolve_issue tool. All compensations are logged in wallet_credits with applied_by = "FoodBot-AI".
FR-804 · Personalised Dish Recommendations
The agent shall analyse a customer's last 10 orders, identify taste patterns (cuisine preference, veg/non-veg, price range), and recommend a dish from a currently-open, highly-rated restaurant the customer has not previously ordered from.
FR-805 · Human Escalation
When the agent cannot resolve an issue (e.g. refund dispute, account problem), it shall create a support_ticket record and inform the customer that a human agent will respond within 2 hours. The admin panel shall display open tickets.
FR-806 · Restaurant Owner Analytics Query
Restaurant owners shall be able to ask the agent questions about their business in plain English (e.g. "Why did my revenue drop this week?"). The agent shall query revenue, orders, and review data, then synthesise a plain-English analysis.
