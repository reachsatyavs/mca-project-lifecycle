Local Development Environment

Making it easy for every team member to run the project locally

The "Works on My Machine" Problem

Without a standardised setup, each team member wastes hours getting the project to run on their laptop. Docker solves this — one docker-compose up command starts the entire stack.

docker-compose.yml for Local Dev

version: '3.8' services: db: image: postgres:15 environment: POSTGRES_DB: fooddelivery POSTGRES_USER: devuser POSTGRES_PASSWORD: devpassword ports: - "5432:5432" volumes: - postgres_data:/var/lib/postgresql/data backend: build: ./backend command: python manage.py runserver 0.0.0.0:8000 volumes: - ./backend:/app ports: - "8000:8000" env_file: - ./backend/.env depends_on: - db frontend: build: ./frontend command: npm run dev volumes: - ./frontend:/app ports: - "3000:3000" volumes: postgres_data:

Quick Start (For New Team Member)

git clone https://github.com/yourteam/food-delivery.git cd food-delivery cp backend/.env.example backend/.env # Edit backend/.env with your keys docker-compose up --build # Backend: http://localhost:8000 # Frontend: http://localhost:3000

This README instruction should work on any team member's machine within 5 minutes. Document this in your README.md.

Staging Deployment (Semester 2)

Deploy to Render or Railway for a shared staging URL. All team members and the faculty guide can access and test the latest version without running it locally.

Auto-Sleep Warning

Render's free tier sleeps after 15 minutes of inactivity and takes ~30 seconds to wake up. For demos, access the URL 2 minutes before your presentation to warm it up. Or use UptimeRobot (free) to ping your app every 14 minutes to prevent sleep.
