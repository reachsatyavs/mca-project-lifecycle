1. README (Initial)

The front door to your project's GitHub repository

A README.md is the first file anyone reads when they visit your GitHub repository. It should answer: What is this? How do I run it? Who built it?

2. SPMP Draft

The project management plan — living document updated each semester

The SPMP (Software Project Management Plan) draft created in Semester 1 contains the full project plan. See the Planning page for the complete SPMP structure. The Semester 1 draft focuses on:

  • Project overview and objectives — what the system is and why it is being built.
  • Scope statement — what is and is not included (links to SRS).
  • Team structure — roles, responsibilities, RACI matrix.
  • Semester 1 deliverables — SRS, TDD, SPMP, PoC.
  • Semester 1–4 roadmap — high-level milestones per semester.
  • Risk register v1.0 — initial list of 6–8 key risks.
  • Communication plan — meeting cadence, reporting to faculty guide.
SPMP Version Control

Store the SPMP as a PDF in the docs/ folder of your repository. Version it: SPMP_v1.0 (Sem 1), SPMP_v2.0 (Sem 2), etc. Each version reflects updates made at the start of that semester based on actual progress.
