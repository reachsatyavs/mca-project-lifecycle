MCA Semester 4 · Compressed Pilot · Bangalore

End-to-End in 13 Weeks
Student Project Programme

A complete project-only semester — from idea to deployed, containerised product. No theory classes. Every week has a deliverable, a gate, and a consultant checkpoint.

Consultant: 2 h/week Docker from Day 1 Git = single source of truth 13 Weeks · Project Only 3 × 2-Week Sprints 2 Consultant Sessions
Consultant Engagement Model — BIMA MCA Sem 4
  • 2 hours/week
  • Review weekly status reports, flag at-risk teams
  • Spot-check 1–2 project artifacts or attend sprint demo
  • Jump in if a team misses 2+ deadlines in a row
  • Faculty: day-to-day execution, demos, grading
  • Consultant: process standards, guide and recovery
Highly Preferred Standards for Project Execution — All BIMA MCA Projects
  • Git repository holds all code + all documentation — no email, no WhatsApp submissions
  • All apps run as Docker containers — Dockerfile + docker-compose.yml committed from Week 1
  • Final demo = docker compose up on a fresh machine — no extra configuration steps

Weeks 3–4 T3 — Design Week 3 — High-level design:
• Architecture pattern (MVC / Client-Server / Microservices — justify choice)
