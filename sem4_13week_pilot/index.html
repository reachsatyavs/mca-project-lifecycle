MCA Semester 4 · Compressed Pilot · Bangalore

End-to-End in 13 Weeks
Student Project Programme

A complete project-only semester — from idea to deployed, containerised product. No theory classes. Every week has a deliverable and a gate.

Docker from Day 1 Git = single source of truth 13 Weeks · Project Only 3 × 2-Week Sprints
Highly Preferred Standards for Project Execution — All BIMA MCA Projects
  • Git repository holds all code + all documentation — no email, no WhatsApp submissions
  • All apps run as Docker containers — Dockerfile + docker-compose.yml committed from Week 1
  • Final demo = docker compose up on a fresh machine — no extra configuration steps
Standard Git Repository Structure — All Teams Follow This
your-project-name/
├── README.md # How to run the project (mandatory)
├── Dockerfile # App container definition
├── docker-compose.yml # All services (app + db + any others)
├── src/ # All application source code
│ └── ...
└── docs/ # All project documentation
├── scope.md # T1 — Project Charter & Scope
├── frd.md # T2 — Functional Requirements
├── design.md # T3 — Software Design Document
├── test-plan.md # Test cases & results
├── final-report.md # T5 — Final Product Document
└── sprints/
├── sprint1.md
├── sprint2.md
└── sprint3.md
13-Week Master Schedule
Weeks Phase Student Deliverables Template
Week 1 T1 — Kickoff & Scope • Form teams (3–4 students), agree on project topic
• Write Project Charter & Scope doc; create Git repo with standard folder structure
• Add Dockerfile stub and docker-compose.yml placeholder
• Define risk register (top 3 risks + mitigation)
• Commit: docs/scope.md + README.md 		T1 — Project Charter →
Week 2 T2 — Requirements • Write FRD: functional requirements (FR-01, FR-02…) + non-functional requirements
• Write User Stories (As a… / I want… / So that…) — minimum 10
• Write Use Cases: actor, pre-conditions, main flow, alt flow
• Write Acceptance Criteria for each story
• Set up Docker skeleton: docker-compose.yml — app + DB containers defined (no code yet)
• Commit: docs/frd.md 		T2 — FRD →
Weeks 3–4 T3 — Design Week 3 — High-level design:
• Architecture pattern (MVC / Client-Server / Microservices — justify choice)
• Component/module breakdown; Docker service map (container ↔ component)
• ER Diagram (all entities, relationships, attributes)
• Level-1 Data Flow Diagram

Week 4 — Detailed design:
• Class diagram (key classes, attributes, methods, relationships)
• 1–2 Sequence diagrams (most important user flows)
• UI wireframes — 2–3 key screens (hand-drawn or Figma/Excalidraw OK)
• Requirements Traceability Matrix — every FR maps to a design component, DB table, class, and test case
• Finalise all test case headers (linked to FRs)
• Commit: docs/design.md (complete) + docs/test-plan.md 		T3 — Design Doc →
Weeks
5 – 10
(3 × 2-week sprints) 		T4 — Build
Sprint 1 · 2 · 3		 Each sprint: plan → build → demo → retrospective

Sprint 1 (W5–6) — core features: At least one working user flow via Docker by W6 Friday
Sprint 2 (W7–8) — secondary features + integration: All modules talking to each other; run test cases
Sprint 3 (W9–10) — polish + stability: No new features. Fix bugs, run full test suite, validate docker compose up on a clean machine. CODE FREEZE end of W10.

Each sprint: daily commits, weekly status (done / doing / blocked), end-of-sprint demo 		T4 — Sprint Plan →
(covers all 3 sprints)
Weeks
11 – 13 		T5 — Harden, Deploy & Ship Week 11 — Deployment hardening:
• Final bug fixes only — no new features
• Validate Docker deployment on a second machine (simulated production)
• Write deployment guide in README (step-by-step fresh install)
• Document environment variables, .env.example, port mapping

Week 12 — Documentation & dry run:
• Complete all sections of T5 Final Product Doc — commit to Git
• Update all diagrams to reflect final code
• Write Lessons Learned + Individual Contribution matrix
• Prepare 10-minute presentation; do a dry run

Week 13 — Final Demo Day:
• 10-min demo to the evaluation panel
• Live demo: docker compose up on evaluator's machine
• Submit Git repo link — all docs + code in repo
• Commit final retrospective 		T5 — Final Doc →

Programme at a Glance
Phase 1 · Weeks 1–2
Define
Scope, requirements, user stories, use cases. Git repo + Docker skeleton live.
Phase 2 · Weeks 3–4
Design
Architecture, ER, DFD, class, sequence, wireframes, RTM. Design gate before any coding.
Phase 3 · Weeks 5–10
Build
3 × 2-week sprints. Code in containers. Demo every sprint. Test as you go.
Phase 4 · Weeks 11–12
Harden & Document
Deploy in Docker, write deployment guide, complete all docs. Dry run demo.
Phase 5 · Week 13
Ship
Final demo, repo submission, retrospective. One command to run the app.
