Week 3: Sections 1–5 Week 4: Sections 6–10 Faculty Sign-off Gate RTM links design to FRD
Design philosophy: Design only what you will build. Every diagram must trace back to the FRD. If a feature isn't in the FRD, it shouldn't be in the design. Keep diagrams simple and correct — a hand-drawn, accurate ER diagram beats a polished but wrong one.

6. Data Flow Diagram — Level 1

Show the main data flows: where data comes from, how it is processed, where it is stored, and what goes back to the user. Level 1 only — no need for level 2 detail.

Level-1 Data Flow Diagram Show: External Entities (users, external systems) → Processes (your app modules) → Data Stores (DB tables)
Label each arrow with the data flowing (e.g. "login request", "book list", "reservation confirmation")
Commit as docs/images/dfd_level1.png

7. Class Diagram (Key Classes Only)

Show the most important classes in your system — typically 4–8 classes. You do NOT need a class for every table. Focus on classes with significant logic (not just data containers).
Class NameKey AttributesKey MethodsRelationships
Class Diagram Draw classes as rectangles divided into 3 parts: Class Name | Attributes | Methods.
Show inheritance (→), composition (◆), and association (—) relationships.
Commit as docs/images/class_diagram.png

8. Sequence Diagrams

Draw 1–2 sequence diagrams for the most important user flows (e.g. login, main feature). Show the exact order of method calls and messages between components.

Sequence Diagram — Flow 1 Participants (top): User → Browser/Frontend → Backend API → Database
Show: each arrow = one message/call, with label. Show return messages too.
Commit as docs/images/seq_login.png
Sequence Diagram — Flow 2 Commit as docs/images/seq_main_feature.png

9. User Interface Wireframes

Show wireframes for 2–3 key screens. Hand-drawn, Figma, Excalidraw, or ASCII — any format is acceptable. What matters is that the layout, key elements, and navigation are clearly shown.

Login Screen Wireframe Show: header, form fields (email, password), submit button, error message area, links (register, forgot password)
Dashboard Wireframe Show: navigation, key content areas, primary actions
Feature Screen Wireframe Show: the main user action your product is built around
