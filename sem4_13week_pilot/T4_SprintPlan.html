Sprint 1 / 2 / 3

Use this template for each of the 3 sprints (Weeks 5–6, 7–8, 9–10). Commit as docs/sprints/sprint1.md (and sprint2.md, sprint3.md).

Sprint 1: Weeks 5–6 Sprint 2: Weeks 7–8 Sprint 3: Weeks 9–10 Demo at end of every sprint
Sprint Rules: (1) Tasks must be small enough to complete in 1–3 days. Break big tasks down. (2) Code runs in Docker from day 1 — no "I'll dockerise later." (3) Daily commits to Git — no big-bang commits on the last day. (4) Every sprint ends with a live demo to faculty.

End-of-Sprint Demo Checklist

Complete this checklist before the demo. A demo that fails due to setup issues is a failed demo. Practice the demo at least once as a team before showing faculty.
  • App starts with docker compose up — no manual steps needed Gate
  • Sprint goal user flow works end-to-end (demonstrated, not just described) Gate
  • No hardcoded credentials or secrets in code (use .env) Gate
  • All sprint backlog tasks updated to Done / In Progress / Carried Over
  • Blockers log updated — all open blockers have owners and action plans
  • Git commits are regular (not one big commit on demo day)
  • Sprint plan committed to docs/sprints/sprint[N].md
  • Team can answer: "What did you not build yet? Why?" without hesitation
