Logging Framework Setup

Capturing structured logs from day one

A logging framework replaces print() statements with structured, configurable log messages that include timestamp, severity, and context.

Log Levels

LevelWhen to UseFood Delivery Example
DEBUGDetailed diagnostic info (dev only)Cart contents calculation steps
INFONormal business eventsOrder ORD-001 placed by user 42
WARNINGUnexpected but handled situationRestaurant R-07 didn't respond in 5 min — auto-cancelling order
ERRORSomething failed but system still runningPayment gateway returned unexpected error code
CRITICALSystem may be going downDatabase connection pool exhausted

Structured Logging (Python / Express.js)

import logging import json logger = logging.getLogger('orders') def place_order(customer, data): logger.info("Order placement started", extra={ "customer_id": customer.id, "restaurant_id": data['restaurant_id'], "item_count": len(data['items']) }) try: order = Order.objects.create(...) logger.info("Order created successfully", extra={ "order_id": order.id, "total": float(order.total) }) return order except Exception as e: logger.error("Order creation failed", extra={ "customer_id": customer.id, "error": str(e) }, exc_info=True) raise

Express.js Logging Configuration

# settings/base.py LOGGING = { 'version': 1, 'disable_existing_loggers': False, 'formatters': { 'json': { '()': 'pythonjsonlogger.jsonlogger.JsonFormatter', 'format': '%(asctime)s %(name)s %(levelname)s %(message)s' }, }, 'handlers': { 'console': {'class': 'logging.StreamHandler', 'formatter': 'json'}, 'file': { 'class': 'logging.handlers.RotatingFileHandler', 'filename': 'logs/app.log', 'maxBytes': 10_000_000, # 10 MB 'backupCount': 5, 'formatter': 'json' }, }, 'root': {'handlers': ['console', 'file'], 'level': 'INFO'}, 'loggers': { 'orders': {'level': 'DEBUG', 'propagate': True}, } }

Error Tracking with Sentry

Automatically capture and report application errors

Sentry is a free error tracking tool that automatically captures unhandled exceptions, groups them by issue, and alerts you by email. It's essential for finding bugs in production that users don't report.

Sentry Setup (Express.js)

pip install sentry-sdk # settings/production.py import sentry_sdk from sentry_sdk.integrations.django import DjangoIntegration sentry_sdk.init( dsn="https://xxxxxxxxxxxx@o0.ingest.sentry.io/0", integrations=[DjangoIntegration()], traces_sample_rate=0.1, # 10% of transactions for performance send_default_pii=False, # Don't send PII (GDPR) environment="production" )

With Sentry, when an order placement fails with an unhandled exception at 2 AM, you get an email immediately — including the full stack trace, user context, and request details. No need to dig through log files.
