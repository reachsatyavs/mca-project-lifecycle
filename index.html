This is a Living, Traceable Document — Not a One-Time Submission

Every document in this reference site is connected to every other document through a deliberate traceability chain: SRS → User Stories → Features → Use Cases → Test Cases → Sprint Items → Change Requests. This chain is built up progressively — Semester 1 captures intent, Semester 2 elaborates into engineering detail, Semester 3 closes the loop with test execution, and Semester 4 finalises with release notes.

Forward Traceability
FR-101 → F-01 → US-C01 → UC-01 → TC-01
If a requirement changes, you immediately know what features, test cases, and sprints are affected.
Backward Traceability
TC-01 → UC-01 → US-C01 → FR-101
Every test case links back to a requirement. No orphan tests. No untested features.
Change Control
Every scope change is a formal CR with impact analysis and faculty sign-off — not a silent code addition that contradicts the SRS.

FoodBot — Agentic AI Customer Support

This project implements a local, privacy-first AI agent powered by Ollama (runs on your own machine — no cloud API, no cost). Unlike a simple chatbot that just generates text, FoodBot is agentic — it reasons, plans, and calls real tools (order DB, restaurant data, wallet credits) to take meaningful actions on behalf of the user.

Ollama + Llama 3.1 ReAct Agent Pattern Tool / Function Calling Node.js + PostgreSQL RAG (Semester 4)
🛵
Customer Support & Order Tracking
"Where is my order? It's been 40 minutes." — Agent checks real-time DB status, estimates delay, offers to cancel or escalate.
Tools used: get_order_status · cancel_order · escalate_to_human
🍽️
Personalised Dinner Recommender
"Suggest something I haven't tried." — Agent analyses 10 past orders, identifies taste patterns, finds a highly-rated open restaurant with a trending dish the customer hasn't ordered.
Tools used: get_order_history · get_open_restaurants · get_trending_dishes
😤
Order Issue Resolution
"My food arrived cold and an item was missing." — Agent investigates delivery time, checks restaurant quality history, decides fair compensation (partial refund, coupon) and applies it.
Tools used: get_order · check_delivery_time · apply_wallet_credit
📊
Restaurant Owner Analytics
"Why did my revenue drop 30% this week?" — Agent queries revenue, ratings, item availability, and competitor approvals, then synthesises a plain-English root-cause analysis.
Tools used: get_revenue · get_reviews · get_item_availability · get_competitors
⏰
Proactive Delay Notifier
No customer action needed — agent scans active orders on a schedule, detects delays, drafts a personalised empathetic message per customer, and adds a compensation coupon when appropriate.
Tools used: scan_delayed_orders · get_partner_location · send_notification · apply_coupon
