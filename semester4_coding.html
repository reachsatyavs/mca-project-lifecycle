1. Production Bug Fix Process

A disciplined hotfix approach that keeps production stable

Fixing bugs in production is different from fixing bugs during development. In production, a bad fix can break a working system. Every fix must be validated on staging before deploying to production.

Hotfix Workflow

# 1. Create hotfix branch from main (NOT develop) git checkout main git checkout -b hotfix/fix-order-minimum-validation # 2. Make the minimal fix — edit only the affected file(s) # Add/update the relevant unit test # 3. Run full test suite locally npx jest --runInBand # 4. Push and open PR to main AND develop git push origin hotfix/fix-order-minimum-validation # 5. Deploy hotfix branch to staging and verify manually # 6. Get code review approval (at least 1 reviewer) # 7. Merge to main → triggers CI/CD to production # 8. Merge to develop → keeps develop branch in sync

Common Semester 4 Bug Categories

Bug TypeExampleFix Approach
Business logic bugDelivery fee not applied to COD ordersFix service layer + add unit test for COD path
UI bugCart total overflows on small screensCSS fix + add responsive test case
Data bugRestaurant rating shows NaN if no reviewsAdd null-safe calculation in rating service
Security patchexpress releases a security update (CVE)npm update express + full regression test
Performance bugAdmin analytics page times out at 30 sAdd DB index + cache the aggregation query
The "Minimal Fix" Principle

In production, make the smallest possible change that fixes the bug. Don't refactor surrounding code. Don't add unrelated features. The larger the change, the higher the risk of introducing new bugs.

2. E2E Test Automation (Playwright)

⭐ Good to Have

Automate full user journeys through the real browser

⭐ Good to Have — Adds significant value to your project report and impresses evaluators. Not required for a passing grade.

End-to-End (E2E) tests drive a real browser through complete user journeys — from login to order placement — and verify that the whole stack works together. They catch bugs that unit and integration tests miss.

What E2E tests cover that unit tests cannot

3. Performance Testing (k6)

⭐ Good to Have

Measure how your API behaves under load before users find out

⭐ Good to Have — Demonstrates production awareness. Even a basic load test with a screenshot in your report is impressive for MCA.

Performance testing checks how your system behaves under concurrent load — multiple users hitting the API simultaneously. It finds bottlenecks before real users do.

Performance Testing Terms

TermWhat it meansExample target
ThroughputRequests handled per second (RPS)> 50 RPS for student project
p95 response time95% of requests complete within this time< 500 ms
Error rate% of requests that return 5xx< 1%
Concurrent usersUsers hitting the system at the same time20–50 for student project

Load Test Script — Restaurant Listing API

// tests/perf/restaurant-list.js import http from 'k6/http'; import { check, sleep } from 'k6'; export const options = { stages: [ { duration: '30s', target: 10 }, // ramp up to 10 users over 30s { duration: '1m', target: 20 }, // stay at 20 users for 1 minute { duration: '20s', target: 0 }, // ramp down to 0 ], thresholds: { http_req_duration: ['p(95)<500'], // 95% of requests under 500ms http_req_failed: ['rate<0.01'], // less than 1% errors }, }; export default function () { const BASE_URL = 'https://your-app.render.com'; const res = http.get(`${BASE_URL}/api/restaurants?city=Mumbai`); check(res, { 'status is 200': (r) => r.status === 200, 'has restaurants array': (r) => JSON.parse(r.body).data.length > 0, 'response time < 500ms': (r) => r.timings.duration < 500, }); sleep(1); // simulate user think time }

Load Test Script — Full Order Flow

// tests/perf/order-placement.js import http from 'k6/http'; import { check, sleep } from 'k6'; export const options = { vus: 10, // 10 virtual users duration: '1m', // run for 1 minute }; const BASE_URL = 'https://your-app.render.com'; export function setup() { // Login once and share the token across VUs const res = http.post(`${BASE_URL}/api/auth/login`, JSON.stringify({ email: 'load-test@test.com', password: 'testpassword' }), { headers: { 'Content-Type': 'application/json' } }); return { token: JSON.parse(res.body).token }; } export default function (data) { const headers = { 'Content-Type': 'application/json', 'Authorization': `Bearer ${data.token}`, }; // Step 1: Browse restaurants const restaurants = http.get(`${BASE_URL}/api/restaurants`, { headers }); check(restaurants, { 'restaurants loaded': (r) => r.status === 200 }); sleep(0.5); // Step 2: Get menu const menu = http.get(`${BASE_URL}/api/restaurants/1/menu`, { headers }); check(menu, { 'menu loaded': (r) => r.status === 200 }); sleep(0.5); // Step 3: Place order const order = http.post(`${BASE_URL}/api/orders`, JSON.stringify({ restaurant_id: 1, items: [{ menu_item_id: 1, quantity: 2 }], delivery_address_id: 1 }), { headers }); check(order, { 'order placed successfully': (r) => r.status === 201, 'order has id': (r) => JSON.parse(r.body).data?.id != null, }); sleep(1); }

⭐ Good to Have

⭐ Good to Have — Takes 1–2 hours but significantly improves code quality and evaluator impression.

Input Validation — express-validator

Ensure all incoming data is validated at the API boundary before it reaches business logic or the database.

// src/validators/order.validator.js const { body, validationResult } = require('express-validator'); const validatePlaceOrder = [ body('restaurant_id').isInt({ min: 1 }).withMessage('Invalid restaurant'), body('items').isArray({ min: 1 }).withMessage('Cart cannot be empty'), body('items.*.menu_item_id').isInt({ min: 1 }), body('items.*.quantity').isInt({ min: 1, max: 20 }), body('delivery_address_id').isInt({ min: 1 }), (req, res, next) => { const errors = validationResult(req); if (!errors.isEmpty()) { return res.status(422).json({ errors: errors.array() }); } next(); } ]; module.exports = { validatePlaceOrder }; // Usage in route: // router.post('/orders', authenticate, validatePlaceOrder, orderController.create);

Rate Limiting — express-rate-limit

// src/middleware/rateLimiter.js const rateLimit = require('express-rate-limit'); // Strict limit for auth endpoints (prevent brute force) const authLimiter = rateLimit({ windowMs: 15 * 60 * 1000, // 15 minutes max: 10, // max 10 attempts per IP message: { error: 'TOO_MANY_ATTEMPTS', message: 'Try again in 15 minutes' }, }); // General API limit const apiLimiter = rateLimit({ windowMs: 60 * 1000, // 1 minute window max: 100, // 100 requests per minute per IP }); module.exports = { authLimiter, apiLimiter }; // In app.js: // app.use('/api/auth', authLimiter); // app.use('/api', apiLimiter);

Graceful Shutdown

Handle SIGTERM properly so in-flight requests finish before the server stops. This is important for zero-downtime deployments.

// src/server.js const app = require('./app'); const { PrismaClient } = require('@prisma/client'); const prisma = new PrismaClient(); const server = app.listen(process.env.PORT || 3000, () => { console.log(`Server running on port ${process.env.PORT || 3000}`); }); // Graceful shutdown process.on('SIGTERM', async () => { console.log('SIGTERM received — closing server gracefully'); server.close(async () => { await prisma.$disconnect(); console.log('Database connection closed'); process.exit(0); }); // Force exit after 10 seconds if still not closed setTimeout(() => process.exit(1), 10_000); });

Part A — RAG: Restaurant Menu Embeddings

RAG (Retrieval-Augmented Generation) solves a key limitation: the LLM doesn't have your restaurant data in its training. Instead of stuffing the entire menu into every prompt (expensive and slow), we pre-compute embeddings of menu items and retrieve only the most relevant ones at query time.

# Install embedding and vector dependencies
npm install @xenova/transformers  # local embedding model (no API key needed)
npm install pg-vector             # pgvector extension for PostgreSQL

# Enable pgvector in PostgreSQL
# Run in psql: CREATE EXTENSION IF NOT EXISTS vector;
// src/ai/rag/embedMenus.js  —  Run this as a one-time script (or nightly cron)
const { pipeline }  = require('@xenova/transformers');
const { prisma }    = require('../../lib/prisma');
const { Pool }      = require('pg');
const pool          = new Pool({ connectionString: process.env.DATABASE_URL });

async function embedMenuItems() {
  const extractor = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2');

  const items = await prisma.menuItem.findMany({
    include: { restaurant: { select: { name: true, cuisineType: true } } }
  });

  for (const item of items) {
    const text = `${item.restaurant.name} | ${item.name} | ${item.description} | ${item.restaurant.cuisineType} | Rs.${item.price}`;
    const output = await extractor(text, { pooling: 'mean', normalize: true });
    const embedding = Array.from(output.data);  // 384-dimensional vector

    await pool.query(
      `INSERT INTO menu_embeddings (menu_item_id, embedding, text_chunk)
       VALUES ($1, $2::vector, $3)
       ON CONFLICT (menu_item_id) DO UPDATE SET embedding = $2::vector`,
      [item.id, JSON.stringify(embedding), text]
    );
  }
  console.log(`Embedded ${items.length} menu items`);
  await pool.end();
}

embedMenuItems();
// src/ai/tools/ragRecommend.js  —  RAG-powered recommendation tool
const { pipeline }  = require('@xenova/transformers');
const { Pool }      = require('pg');
const { prisma }    = require('../../lib/prisma');
const pool          = new Pool({ connectionString: process.env.DATABASE_URL });

const ragRecommendTool = {
  definition: {
    type: 'function',
    function: {
      name: 'rag_recommend_dishes',
      description: 'Recommend dishes based on customer taste profile using semantic search over restaurant menus.',
      parameters: {
        type: 'object',
        properties: {
          customerId: { type: 'number' },
          tasteQuery:  { type: 'string', description: 'Natural language description of what the customer might enjoy, e.g. "spicy South Indian vegetarian"' }
        },
        required: ['customerId', 'tasteQuery']
      }
    }
  },

  async execute({ customerId, tasteQuery }) {
    const extractor = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2');
    const output    = await extractor(tasteQuery, { pooling: 'mean', normalize: true });
    const queryVec  = JSON.stringify(Array.from(output.data));

    // Cosine similarity search — get top 5 semantically matching menu items
    const { rows } = await pool.query(
      `SELECT me.text_chunk, mi.name, mi.price, r.name AS restaurant, r.rating, r.is_open,
              1 - (me.embedding <=> $1::vector) AS similarity
       FROM menu_embeddings me
       JOIN menu_items mi ON me.menu_item_id = mi.id
       JOIN restaurants r ON mi.restaurant_id = r.id
       WHERE r.is_open = true
       ORDER BY me.embedding <=> $1::vector
       LIMIT 5`,
      [queryVec]
    );

    return { recommendations: rows.map(r => ({
      dish: r.name, price: r.price, restaurant: r.restaurant,
      rating: r.rating, similarity: Math.round(r.similarity * 100)
    })) };
  }
};

module.exports = { ragRecommendTool };

Part B — Context Compression (Long-Session Memory)

When a conversation exceeds 20 messages, rather than dropping old context, we ask the LLM to summarise the conversation so far into a compact paragraph. This summary replaces all old messages, keeping the context window small while preserving important facts.

// src/ai/contextManager.js
const Ollama = require('ollama').default;
const { prisma } = require('../lib/prisma');
const ollama = new Ollama({ host: 'http://localhost:11434' });

const MESSAGE_LIMIT = 20;

async function getContextMessages(sessionId) {
  const messages = await prisma.aiMessage.findMany({
    where: { sessionId },
    orderBy: { createdAt: 'asc' }
  });

  if (messages.length <= MESSAGE_LIMIT) {
    return messages.map(m => ({ role: m.role === 'tool' ? 'tool' : m.role, content: m.content }));
  }

  // Messages exceed limit — compress older ones
  const toCompress = messages.slice(0, messages.length - 10);
  const recent     = messages.slice(messages.length - 10);

  const summaryPrompt = `Summarise this conversation briefly (3-4 sentences), preserving key facts like order IDs, issues raised, compensations applied, and customer preferences:\n\n${toCompress.map(m => `${m.role}: ${m.content}`).join('\n')}`;

  const { message } = await ollama.chat({
    model: 'llama3.1:8b',
    messages: [{ role: 'user', content: summaryPrompt }]
  });

  // Save compressed summary to session
  await prisma.aiConversationSession.update({
    where: { id: sessionId },
    data: { contextSummary: message.content }
  });

  return [
    { role: 'system', content: `Earlier conversation summary: ${message.content}` },
    ...recent.map(m => ({ role: m.role === 'tool' ? 'tool' : m.role, content: m.content }))
  ];
}

module.exports = { getContextMessages };

Part C — Proactive Delay Notifier Agent

This agent runs as a background scheduled job (every 5 minutes). It needs no customer to initiate a conversation — it scans for delayed orders, drafts personalised messages, and sends notifications with compensation coupons where appropriate.

// src/ai/proactiveAgent.js  —  Run via: node-cron every 5 minutes
const Ollama   = require('ollama').default;
const { prisma } = require('../lib/prisma');
const { resolveIssueTool } = require('./tools/resolveIssue');
const ollama = new Ollama({ host: 'http://localhost:11434' });

async function runDelayNotifierAgent() {
  // 1. Find orders that are delayed (delivered_at > estimated_delivery_at + 15 min)
  const delayed = await prisma.order.findMany({
    where: {
      status: 'OUT_FOR_DELIVERY',
      estimatedDeliveryAt: { lt: new Date(Date.now() - 15 * 60 * 1000) },
      delayNotificationSent: false
    },
    include: { customer: true, restaurant: true, orderItems: { include: { menuItem: true } } }
  });

  for (const order of delayed) {
    const delayMins = Math.round((Date.now() - order.estimatedDeliveryAt.getTime()) / 60000);

    // 2. Ask LLM to draft a personalised, empathetic message
    const { message } = await ollama.chat({
      model: 'llama3.1:8b',
      messages: [{
        role: 'user',
        content: `Write a short, empathetic delay notification (2-3 sentences) for customer ${order.customer.name}.
Their order from ${order.restaurant.name} is ${delayMins} minutes late.
Items ordered: ${order.orderItems.map(i => i.menuItem.name).join(', ')}.
Mention that a compensation coupon has been added. Be warm and apologetic, not robotic.`
      }]
    });

    // 3. Apply compensation credit
    const creditAmount = delayMins > 30 ? 100 : 50;
    await resolveIssueTool.execute({
      customerId: order.customerId,
      orderId: order.id,
      amount: creditAmount,
      reason: `Proactive compensation for ${delayMins}-minute delivery delay`
    });

    // 4. Send notification (via push/SMS service)
    await prisma.notification.create({
      data: {
        userId: order.customerId,
        type: 'DELAY_PROACTIVE',
        message: message.content,
        metadata: { orderId: order.id, delayMins, creditAmount }
      }
    });

    // 5. Mark so we don't notify twice
    await prisma.order.update({
      where: { id: order.id },
      data: { delayNotificationSent: true }
    });

    console.log(`[ProactiveAgent] Notified customer ${order.customerId} for order ${order.id} — ${delayMins}min delay`);
  }
}

Part D — AI Performance Monitoring

A simple monitoring endpoint so you can track the health of the AI feature without a third-party tool — essential for your project viva.

// src/routes/aiMonitor.js  (admin only — GET /api/admin/ai/metrics)
const express = require('express');
const router  = express.Router();
const { prisma } = require('../lib/prisma');
const { authenticateToken, requireRole } = require('../middleware/auth');

router.get('/metrics', authenticateToken, requireRole('ADMIN'), async (req, res) => {
  const [totalSessions, openTickets, totalMessages, toolStats, avgTokens] = await Promise.all([
    prisma.aiConversationSession.count(),
    prisma.supportTicket.count({ where: { status: 'open' } }),
    prisma.aiMessage.count(),
    prisma.aiMessage.groupBy({
      by: ['toolName'],
      where: { toolName: { not: null } },
      _count: { toolName: true }
    }),
    prisma.aiMessage.aggregate({ _avg: { tokensUsed: true } })
  ]);

  const escalationCount = await prisma.supportTicket.count();
  const resolvedByAI    = await prisma.walletCredit.count({ where: { appliedBy: 'FoodBot-AI' } });

  res.json({
    totalSessions,
    totalMessages,
    openSupportTickets: openTickets,
    escalationRate: totalSessions ? `${((escalationCount / totalSessions) * 100).toFixed(1)}%` : '0%',
    issuesResolvedByAI: resolvedByAI,
    avgTokensPerMessage: Math.round(avgTokens._avg.tokensUsed || 0),
    toolCallBreakdown: Object.fromEntries(toolStats.map(t => [t.toolName, t._count.toolName]))
  });
});

module.exports = router;
