UI Wireframes & Screen Mockups

Example Project: Online Food Delivery System

Wireframes show the layout, content hierarchy, and user flows before any real code is written. These high-fidelity mockups represent the key screens of the Online Food Delivery app — built with HTML/CSS to match what the final React/Vue frontend should look like.

Design System Notes

Colours, typography, and component guidelines

ElementValueUsage
Primary colour#FF6B35Buttons, CTAs, active states, nav bar
Dark background#1A1A2EMenu headers, owner dashboard nav, headings
Success green#2E7D32Ratings, veg indicator, accepted status
Error red#C62828Non-veg indicator, reject button
Font stack-apple-system, Segoe UI, RobotoBody text; system font for performance
Base font size14px app / 12px secondaryReadable on 375px+ screens
Border radius8px cards / 20px chips / 50% avatarsModern soft corners throughout
Shadow0 1px 4px rgba(0,0,0,0.06)Cards — subtle depth without heavy shadows
Handoff to Frontend Developer

In Sprint 5, share these wireframes as the frontend spec. For each screen, document: (1) API endpoint the screen calls, (2) state that triggers re-render, (3) loading/error states, (4) responsive breakpoints. Use Figma to add real assets (photos, icons) on top of this HTML mockup.
