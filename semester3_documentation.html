Documentation — User Manual, Test Reports & Deployment Guide

Example Project: Online Food Delivery System

Semester 3 documentation completes the user-facing and operational documentation — the user manual, test reports from all testing activities, and a deployment guide for reproducing the production environment.

📖

User Manual

Step-by-step guide for each type of user

The User Manual is written for non-technical users — people who will use the system but didn't build it. It covers every screen and operation with screenshots.

User Manual Structure

ChapterTopics
1. Customer GuideRegistration, login, browsing restaurants, filtering by cuisine, viewing menus, adding to cart, checkout, tracking order, viewing history, leaving reviews, managing profile & addresses
2. Restaurant Owner GuideRegistration, awaiting admin approval, setting up menu (add/edit/delete items, upload photos), managing incoming orders (accept/reject), viewing daily order summary
3. Delivery Partner GuideRegistration, going online/offline, viewing assigned orders, marking pickup and delivery
4. Admin GuideLogging in, approving/rejecting restaurant applications, managing users, viewing platform reports
5. TroubleshootingCommon issues: payment failed (retry steps), order not accepted (auto-cancel rules), login issues (password reset steps)
Screenshot Tip

Take screenshots of every screen in the user manual. Use a tool like Snagit or simply macOS's built-in screenshot tool. Annotate screenshots with numbered callouts pointing to buttons. A user manual with screenshots is 10× easier to follow than one without.

Deployment Guide

How to reproduce the production environment from scratch

The Deployment Guide enables anyone to deploy the system to a fresh server. It is essential for grading (faculty may try to deploy it) and for the Semester 4 production launch.

Deployment Guide Sections

  1. Prerequisites — Python 3.11+, Docker, Git, a Render account, a Supabase account.
  2. Environment Variables — Full list of all required .env variables with descriptions (not values).
  3. Database Setup — Create Supabase project, copy connection string, run migrations.
  4. Deploy Backend to Render — Step-by-step: connect GitHub repo, set env vars, deploy.
  5. Deploy Frontend to Vercel — Import from GitHub, set VITE_API_BASE_URL.
  6. Post-Deploy Steps — Create superuser, seed initial data, verify health endpoint.
  7. Verify Deployment — Go-live checklist items to run after every deployment.
